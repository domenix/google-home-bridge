1 · Public endpoint

Google must reach this instance over HTTPS. Current mode: (set in the add-on Configuration tab)

2 · Home Assistant proxy trust

Add this to configuration.yaml so Home Assistant trusts the bridge (once, then restart):

http:
  use_x_forwarded_for: true
  trusted_proxies:
    - 172.30.32.0/23

3 · Google project

  1. In the Google Home Developer Console, Create a project, then add a Cloud-to-cloud integration.
  2. Copy the project ID from the console URL here — the values below are built from it:
  1. Fill the console form top to bottom:
Integration nameanything, e.g. Home Assistant
Device typeany — e.g. Control Bridge
App icon144×144 transparent PNG, file name = project ID
OAuth Client ID
Client secretanything (unused by Home Assistant)
Authorization URL
Token URL
Cloud fulfillment URL

Leave the rest at defaults — Local fulfillment off, no scopes, basic-auth checkbox unchecked — and Save.

4 · Service account (report state)

  1. In Google Cloud Console, select the same project and enable the HomeGraph API.
  2. Create a service account — name it anything; skip both optional steps (no role, no principals), click Done.
  3. Open the new account → Keys → Add key → Create new key → JSON, then paste the downloaded file below.

5 · Link & go

  1. In the Google Home app: + Add device → Works with Google Home and pick [test] your project name.
  2. Log in with your Home Assistant credentials — devices sync automatically.
  3. Pick which entities Google sees under Settings → Voice assistants → Expose, exactly like Nabu Casa.
